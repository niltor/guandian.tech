为了快速总结上面发生的情况, 代码执行以下操作:

  1. PowerShell 控制台被告知将 CTRL C 视为输入, 而不是终止操作。
  2. 1秒的睡眠执行 (以确保键盘中键入的所有内容) 后跟键盘缓冲区刷新只是为了确保在执行 while 循环之前未按下任何键。
  3. while 循环构造开始时, 您可以在脚本中放置任何要签入的条件, 脚本将在其中检查是否存在任何挂起或当前正在运行的作业。如果没有, 则脚本自然退出。
  4. 每个 while 循环迭代的第一部分是检查是否有任何键被按下 (和释放), 如果有它检查是否是字符 "3", 这是等效的 CTRL C。
  5. 如果是这样, 脚本通知用户它看到执行的操作, 然后继续正常关闭任何正在运行的作业。
  6. 在作业关闭后, PowerShell 控制台被告知要像正常一样对待 CRTL, 然后调用一个函数, 以便在执行其他清理时正常退出脚本。

您可以为所需的任何循环构造执行此代码并替换 while 循环, 使用您希望保持循环运行的任何条件, 并使用 CTRL-c 检测来执行所需的任何清理操作。例如, 您可以使用 ForEach 循环构造以某种方式处理100K 邮箱, 如果按下 ctrl 键, 则可以转储到文件中执行的操作, 直到按下 ctrl c 的点, 而将其全部丢失, 这是 PowerSh 的默认操作。ell.

关闭的想法

希望在 PowerShell 控制台控件上的这个小的趣闻可以帮助你与任何大/长运行的脚本, 你不希望倾倒所有收集/执行/运行在后台-部分方式通过脚本的执行。

我在许多短时间运行的脚本中不使用此项, 但我将返回重新评估以前发布的脚本, 以查看它们是否可以从这种类型的控件中受益。
